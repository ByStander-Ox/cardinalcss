Cardinal

Build more with less.

View on Github // Download

Cardinal is a barebones, “mobile-first” front-end web development framework with some useful default styles, responsive layout and typography, and a simple grid system.

Who should use this?

If you are developing a responsive web application, then Cardinal is your friend. This framework tosses the idea of "pixel perfect" web design mostly out the window, instead placing more importance on good layout and readability for a wide range of devices.

Cardinal uses a combination of modular scale, unitless line-heights, and REM units to provide typography and layout that is not only readable, but is also easy for designers and developers to maintain and manipulate.

Instead of a traditional CSS reset, Cardinal includes a minified version of normalize.css by Nicolas Gallagher to help render HTML elements more consistently across modern browsers, as well as Internet Explorer 8 and above.

Base

box-sizing: border-box;

Cardinal declares some useful default styles to the <html> and <body> elements to get them to play nice across browsers. It also adds basic styles to text selections, and applies box-sizing: border-box; to every element to fixed the flawed CSS box model.

Fonts

@font-face {}

To make adding custom fonts a bit quicker, Cardinal includes an example code snippet of the bulletproof @font-face declaration, so that you don’t have to worry about looking it up every time you want to use custom fonts in your project.

Typography

Following a "mobile-first" approach, Cardinal first sets the root font-size on the <html> element to 75%, or 12px, and provides a fallback for IE8 and below to set the root font-size to 100%, or 16px.

html { font-size: 75%; }
html.lt-ie9 { font-size: 100%; }

The root font-size is 12px because its optimized for feature phones (mobile devices with a viewport width less than 320px). This root font-size can then be overwritten with subsequent media queries, depending on the minimum width of the browser viewport.

@media only screen and (min-width: 320px) { html { font-size: 81.25%; } }
@media only screen and (min-width: 480px) { html { font-size: 87.5%;  } }
@media only screen and (min-width: 768px) { html { font-size: 100%;   } }
@media only screen and (min-width: 960px) { html { font-size: 112.5%; } }

Non-typography

There are also some sensible default styles for non-typographical elements included with Cardinal. For instance, all <img> and <video> tags are given fluid properties, and <textarea> elements are restricted to vertical resizing only.

img,
video {
	max-width: 100%;
	width: auto\9;
	height: auto !important;
}
textarea {
	resize: vertical;
}

Grids

Based on the clever ideas of Matt Berridge, Cardinal’s grid system uses fluid item widths and fixed gutters. Grid items can be divided into halves, thirds, fourths, fifths, or sixths. Nesting grid elements is a breeze, and you can dictate the grid’s behavior by adding CSS classes to items within the grid.

The item widths are set in percentages, and the gutters are set in REMs. Due to the change in the root font-size above, the fixed gutters increase/decrease proportionally, depending on the width of the viewport.

The following example can be described as: “a grid with four 100% width items, which become 50% width items on lap devices, and 25% width items on desktop devices.”

Usage

  • .grid - makes element a grid wrapper.
  • .grid_item - makes element a grid item.
  • .one_half,
    .two_thirds, etc - makes element a specific width of its parent.
  • .lap-* - grid breakpoint for devices you hold in your lap.
  • .desk-* - grid breakpoint for desktop devices.
<div class="grid">
	<div class="grid_item lap-one_half desk-one_fourth"></div>
	<div class="grid_item lap-one_half desk-one_fourth"></div>
	<div class="grid_item lap-one_half desk-one_fourth"></div>
	<div class="grid_item lap-one_half desk-one_fourth"></div>
</div>

Layout

Cardinal also includes some reusable classes for common layout modules, like page wrappers, content wells, and 100% width containers.

.wrapper {
	width: 90%;
	max-width: 1280px;
	margin: 0 auto;
}

	margin: 0;
	margin-bottom: 1.333rem;
	padding: 0;
	background: #efefef;
	list-style: none;
}

	padding: 1.333rem;
	-webkit-border-radius: 5px;

}

Prompts

By default, Cardinal includes two small prompts, one that shows up for users who have Javascript disabled, and one for the IE6 Chrome Frame. These are hidden by default unless the user has Javascript disabled, or is using IE6 and below.

Helpers

A lot of helpers from the fantastic HTML5 Boilerplate are used here, along with some custom helpers for quick text alignment, margins, paddings, and floats.
