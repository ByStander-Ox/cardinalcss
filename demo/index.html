Cardinal

Build more with less.

View on GitHub Download

What gives?

Cardinal is a barebones, responsive front-end web development framework equipped with bare minimum styles, zoomable typography, and a simple fluid grid system.

This framework takes the "mobile first" approach to front-end web development, pulling from incredible resources like HTML5 Boilerplate, normalize.css, jQuery, Modernizr, and others to provide an excellent base to start your front-end development work.

Minimal styling

While frameworks like Bootstrap and Foundation provide many useful tools and look beautiful, their signature styling has become quite ubiquitous. Cardinal tries to stay out of your way by only providing the bare minimum, leaving the creativity up to you.

Rich typography

Don't underestimate the importance of great type on the web. Cardinal combines the structure of a baseline grid with the concept of modular scale to provide a strong vertical rhythm. Images and videos will inevitably throw off the baseline grid, but that's okay.

Simple, fluid grid

Cardinal's dead simple, content-driven fluid grid system will have you creating awesome layouts in no time. Nesting is a cinch, too! Big ups to Paul Irish for applying box-sizing: border-box; on all the thingrid.

Grid system

Based on the clever ideas of Matt Berridge, Cardinal's grid system uses the powerful CSS3 box-sizing property along with fluid (%-based) column widths and fixed (rem-based) column gutters. Nesting grid elements is a breeze, and you can dictate the grid's behavior by adding custom media query breakpoints and CSS classes to your columns.

<section class="grid_divider">
	<div class="grid_wrapper">
		<div class="grid_col grid_bp1_col-one_fourth"></div>
		<div class="grid_col grid_bp1_col-one_fourth"></div>
		<div class="grid_col grid_bp1_col-one_fourth"></div>
		<div class="grid_col grid_bp1_col-one_fourth"></div>
	</div>
</section>

Usage

By default, the grid is split into four columns, or quarters. If you need more columns, simply add more classes to the system.

Required classes

Every column must have the .grid_col class, and a set of columns needs to be wrapped with a <div class="grid_wrapper"> element.

  • .grid_wrapper - contains a set of grid columns
  • .grid_col - designates an element as a column in the grid
  • .grid_col-* - determines a column's proportion (for example, "grid_col-one_third" sets a column to one-third the width of its container)

Optional classes

  • .grid_bp*_col-* - splits a grid wrapper into columns based on the specified breakpoint (.bp1, .bp2, etc), or @media query where that class lives.
  • .grid_divider - adds a border to the bottom of a grid wrapper (.grid_wrapper)
  • .grid_gutter-half - Add this class to the .grid_wrapper element to get column gutters that are half the default size.

Example: Simple grid, custom breakpoints

Now we're talking. This grid includes columns that are aware of the custom breakpoints defined in this project. For instance, using the .grid_bp1_col-one_half class on a column means that when the viewport reaches a minimum width equal to the first @media query breakpoint, change that column's proportion to one half. The *-one_half suffix in the class name stands for the proportion. In this case, it's one-half. Make sense?
